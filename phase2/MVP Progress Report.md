Objective and Current MVP Approach
#
The final MVP will use an AI-driven system to help everyday investors interpret social media posts made by politically or financially influential people and understand how those posts may relate to the movement of the stock market. Being able to understand whether a post is meaningful, what type of market signal it may contain, and whether it may or may not influence an investor's financial decisions is a very difficult skill to master and is impossible to effective when a human can only read so many posts in a given timeframe. With the prevalence of social media, many important public figures communicate their strong opinions, intentions, or hints at policy changes through short online posts. For the average investor, it is extremely difficult to manually track this information to judge its relevance and analyze the effect it may or may not have on market behavior.

The current approach for the final MVP focuses on creating a small-scale prototype that addresses the earliest stages of that pipeline, converting text into structured numerical data. During this early stage of development, the system will not yet make real market predictions from live political posts, nor does it analyze a full portfolio. Instead, the MVP demonstrates that public statements can be processed automatically and transformed into a data matrix that a computer can later use for mathematical modeling. In its current form, the prototype reads text from a file, separates it into blocks, and produces basic measurable outputs such as word count, emotional-word count, and sentiment score. This MVP is intentionally narrow so the team can validate the text-analysis concept before expanding into data collection, stock-price correlation, and forecasting.
#
What Has Been Built So Far
#
Currently, a working text analysis prototype has been developed in Python. The program takes a text file as input, reads the contents, separates the text into multiple blocks using blank lines, and analyzes each block individually. For every block of text, the prototype calculates the total number of words, extracts a short list of key phrases using TextBlob, estimates how many words are emotionally charged using the VADER lexicon, and assigns a sentiment score using a pretrained transformer sentiment model. These outputs are then stored as data points and assembled into a numerical matrix that can be used later for modeling or comparison.

This proves that qualitative language can be converted into quantitative features. Rather than leaving posts as raw text, the system begins translating them into a form that can be compared, plotted, and eventually correlated with stock movement. The sample output demonstrates that different text blocks generate different feature values, meaning the program is already detecting variation in sentiment and structure. Even though the test inputs are simple and not yet financial or political in nature, the prototype establishes the underlying workflow that future versions of the MVP will rely on: input text, analyze content, extract features, and output machine-readable data.

Technical Approach

The technical approach used in this phase combines several lightweight natural language processing tools into a single prototype pipeline. The project is written in Python and uses widely available libraries to perform different parts of the text analysis task. VADER is used as a lexicon-based sentiment resource to identify emotionally charged words, while a Hugging Face transformer model is used to assign an overall positive or negative sentiment score to each text block. TextBlob is used to identify noun phrases, which helps capture keywords or important subjects discussed in the text. NumPy is then used to organize the numerical results into matrix form, making the output easier to pass into future analysis steps.

This approach was chosen because it allows the team to build a functioning proof of concept without first training a large custom model. At this stage, the goal is not to outperform production-grade financial language models, but to demonstrate that a layered text analysis process can generate useful features from social media-style language. By combining simpler counts, lexicon features, model-based sentiment, and keyword extraction, the prototype already moves beyond a purely rule-based system. It also creates a foundation for later work, where these text-derived features can be aligned with stock data, time windows, and potentially more advanced models that better capture context, tone, and financial significance.

Evidence of Progress

The current prototype is a Python-based text processing program that converts raw text into a categorized data matrix. It accepts a .txt file as command-line input, reads the file contents, and partitions the text into separate blocks using blank-line delimiters. Each block is then passed through several natural language processing components. A token-based word count is computed first, followed by an emotional-word count using the VADER sentiment lexicon. The text is also evaluated with a pretrained transformer sentiment-analysis model, which returns a polarity label and confidence score that the program converts into a signed sentiment value. In parallel, TextBlob is used to extract noun phrases that serve as lightweight keyword features. For each text block, the program stores these outputs as a single data point, then assembles the numeric features into a NumPy matrix for downstream analysis.

This progress is essential for the MVP because the full system depends on transforming public statements into machine-readable variables that can be compared against stock-market behavior. Raw social media posts are unstructured text, so they must first be encoded into measurable features before any correlation, regression, or predictive modeling can occur. The current prototype establishes that feature-extraction stage by generating structured sentiment and topic-related data from text automatically. This provides the foundation for the next MVP steps, where these extracted features can be aligned with time-series market data and used to analyze how specific public statements may correlate with movement in selected stocks or broader indices.

Current Limitations and Open Risks

This prototype does not yet collect real data from X or any other live source, so all testing must be done using manually prepared text files. That means the prototype has not yet been tested on the noisy, inconsistent, and often abbreviated style of real social media posts. Second, the current features are relatively simple and may not capture the true market relevance of a statement. A post can be emotionally negative but economically unimportant, or it can be neutral in tone while containing extremely significant policy information. Sentiment alone is not enough to determine impact on stock prices.

There are also broader project risks that remain unresolved. One major risk is data availability, especially if access to public posts is limited by platform restrictions, API costs, or scraping constraints. Another risk is the challenge of proving meaningful correlation rather than random association between posts and market movement. Financial markets are influenced by many variables at once, so even a strong-looking relationship may not be predictive in practice. A third risk is evaluation: the team still needs a clear way to measure whether the system is actually helping users make better interpretations or predictions. These limitations highlight the work still required before the project can become a reliable MVP.

Plan for Phase 3

In Phase 3, the focus should shift from isolated text analysis to actual market behavior correlation. The next major step is to build a program that uses real-world data rather than hand-written sample text. This means collecting public posts from selected influential individuals over a defined time period and pairing those posts with stock market data, such as S&P 500 values or individual stock prices. Once both sources are available, the team can align text features with market movement over time and begin testing whether certain types of posts correlate with short-term or medium-term market responses. Additional text features should also be added, such as topic labels, named entities, post frequency, and time-based clustering, so that the analysis becomes more informative than simple sentiment alone.

Phase 3 should also include work on usability and interpretation, since the long-term purpose of the system is to help a user make sense of important online statements. That means the MVP should move toward a user-facing format that displays not only a prediction, but also an explanation of why that prediction was generated. For example, the system could show which people were analyzed, which recent posts contributed most strongly to the model, and whether the posts were interpreted as positive, negative, or uncertain. At the same time, the team should begin validating model quality by comparing predictions against actual historical outcomes across multiple test periods. The goal for Phase 3 is not necessarily a perfect market predictor, but a working MVP that demonstrates end-to-end functionality: gather posts, analyze text, connect it to market data, and present the results in a way that is understandable and useful.
